HTML

  • HTML creates the structure of a website.
  • Head elements contain metadata that is not visible to the user, but helps render the webpage correctly.
  • There are 'title' 'style' 'base' 'link' 'script' and 'meta' tags that can be used inside the head element.
  • Body elements contain content that is shown to the user.
  • The meta viewport tag should be included in the head of all websites to ensure it renders correctly on all devices.
  • There are 'parent' and 'child' elements.
  • Many HTML elements are semantic (they are named what they represent) and it's good practice to use these.
  • H1 to H6 should be used in order on a page, h1 should only be used once.

CSS

  • Three ways to do CSS: inline, internal style sheet, external style sheet.
  • The external style sheet needs to be linked to your HTML by using the 'link' tag in the 'head'.
  • Margin property is the space inside an element.
  • Padding property is the space outside an element.
  • Think of each element as a box, with content, padding, margin.
  • Selectors denote which elements you want the CSS rules to apply to.
  • The selector * (asterix) applies the rule to ALL visible elements on a document.
  • You can apply multiple selectors to a single line of code by separating them with a comma, so that you can apply the same formatting to multiple elements (eg. header, footer) without repeating code.
  • Eg: img is a selector, block is a value (which assigns each image to start on a new line).
  • No need to use triangular brackets.
  • Class attributes: allows you to apply CSS to certain elements only.
  • The full stop before a class attribute (eg .class) denotes that this is the class selector, so any element assigned to 'card' will have this formatting.
  • In the HTML, this will look like section class="card".

Git

  • All git commands must start with git.
  • git status: checks what branch we are currently on.
  • git checkout -b feature/new-branch-name: creates new branch.
  • 1. git add -A: stage all changes in a file (ready to be committed).
  • 2. git commit -m "explain-changes-being-committed": commit your code with a message to explain what changes you are committing.
  • 3. git pull origin main: pull the latest version of the branch on to your local machine (check you're up to date basically).
  • 4. git push origin feature/featurename: pushes changes up to GitHub ready for review and merging with main branch.

JavaScript

  • Variables are containers that store values.
  • JavaScript is case sensitive.
  • Semicolons indicate where a statement ends.
  • There are different data types, eg. a string (text), number, boolean (true/false), array, object (can be anything!).
  • Comments can be added to JS with /*comment*/ - this will 'hide' the code.
  • An operator is a mathematical symbol that produces a result, such as adding, assigning, strict equality.
  • To declare a variable, we use 'var' and then give the variable a name.
  • The console.log() method instructs content inside the brackets to be logged to the console (which you can access by right clicking and hitting inspect in google
    • Control flow is the order in which computer executes code or text comment.
  • IF statements check if a condition is truthy or falsy. The code will then only be executed if the condition is truthy.
  • A value is falsy if it is: 0, -0, 0n, "", null, undefined, NaN, or false.
  • Use 'else if' to go through more conditions.
  • ARRAYS: allows you to assign multiple values to a variable, using []. These are indexed, starting from 0, so that you can pull out a single value from an array.
  • FOR LOOPS: allows you to execute code multiple times. You must determine the starting point for the loop, give the condition (eg. if the number is less than the length of the array, continue), and then put in code to allow the array to iterate over each item.
  • FUNCTIONS: instruct the computer to perform a task. You must define the function, and then call it. Functions are not automatically executed like for/if statements.
  • Functions are defined by using the keyword 'function', and the name of the function must always be followed by parentheses ().
